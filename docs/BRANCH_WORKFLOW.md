## 📋 Branch Naming Conventions

### 1. Claude Code Sessions
**Pattern:** `claude/<description>-<session-id>`

The session ID is automatically generated by Claude Code and ensures branch isolation.

**Examples:**
```
claude/add-voice-streaming-01ABC123XYZ
claude/fix-auth-bug-02DEF456GHI
claude/refactor-api-03JKL789MNO
```

**Guidelines:**
- Use lowercase with hyphens
- Keep description short and clear (2-4 words)
- Session ID is auto-appended

### 2. Feature Branches
**Pattern:** `feature/<feature-name>`

For larger features developed outside Claude Code sessions.

**Examples:**
```
feature/voice-cloning
feature/stripe-payments
feature/admin-dashboard
feature/real-time-streaming
```

### 3. Bug Fixes
**Pattern:** `fix/<issue-description>`

For bug fixes and small corrections.

**Examples:**
```
fix/supabase-auth-redirect
fix/mobile-login-timeout
fix/api-rate-limiting
```

### 4. Hotfixes
**Pattern:** `hotfix/<critical-issue>`

For urgent production fixes.

**Examples:**
```
hotfix/payment-webhook-failure
hotfix/database-connection-leak
```

### 5. Release Branches
**Pattern:** `release/<version>`

For preparing releases (if using versioning).

**Examples:**
```
release/v1.0.0
release/v1.1.0-beta
release/v2.0.0
```

## 🔄 Branch Lifecycle

### 1. Creating a Branch

**From Claude Code:**
Claude automatically creates branches with the pattern `claude/<description>-<session-id>`.

**Manual creation:**
```bash
# For a feature
git checkout -b feature/voice-cloning main

# For a bug fix
git checkout -b fix/auth-redirect main

# Always branch from main (unless specified otherwise)
```

### 2. Working on a Branch

```bash
# Make changes
git add .
git commit -m "Add voice cloning API endpoint"

# Push regularly
git push -u origin feature/voice-cloning
```

**Commit Message Format:**
- Use present tense: "Add feature" not "Added feature"
- Be descriptive but concise
- Reference issues if applicable: "Fix auth redirect (closes #42)"

### 3. Creating a Pull Request

**Before creating PR:**
```bash
# Ensure you're up to date with main
git fetch origin main
git rebase origin/main

# Run tests
npm test  # or your test command

# Push
git push -u origin your-branch-name
```

**PR Template:**
```markdown
## Summary
Brief description of changes

## Changes
- Change 1
- Change 2
- Change 3

## Test Plan
How to test this PR

## Screenshots (if applicable)
[Add screenshots]

## Checklist
- [ ] Tests pass
- [ ] Code follows style guidelines
- [ ] Documentation updated
- [ ] No breaking changes (or noted in description)
```

### 4. Merging

**Options:**
1. **Squash and merge** (recommended for feature branches)
   - Keeps main history clean
   - All commits squashed into one

2. **Merge commit** (for significant features)
   - Preserves full history
   - Shows merge point in graph

3. **Rebase and merge** (for small fixes)
   - Linear history
   - No merge commit

**After merging:**
```bash
# Switch to main
git checkout main

# Pull latest
git pull origin main

# Delete local branch
git branch -d feature/voice-cloning

# Delete remote branch (if not auto-deleted by GitHub)
git push origin --delete feature/voice-cloning
```

### 5. Archiving

Branches are archived in two ways:
1. **Pull Request history** - Even after branch deletion, PR remains
2. **Backup file** - `ARCHIVED_BRANCHES_YYYYMMDD.txt` contains all branch info

**To create a backup:**
```bash
git branch -r --format='%(refname:short)|%(committerdate:short)|%(subject)' > ARCHIVED_BRANCHES_$(date +%Y%m%d).txt
```

## 🧹 Branch Cleanup

### When to Delete Branches

**Delete immediately after merge:**
- Feature branches
- Bug fix branches
- Claude Code session branches

**Keep for reference:**
- Release branches
- Long-running experiment branches (mark with `experiment/` prefix)

### Automated Cleanup Script

Run the cleanup script to remove old merged branches:

```bash
cd premier_voice_assistant
./scripts/cleanup_old_branches.sh
```

**What it does:**
1. Creates backup of all branch information
2. Lists branches to be deleted
3. Asks for confirmation
4. Deletes merged branches from remote
5. Reports success/failure

### Manual Cleanup

**Find merged branches:**
```bash
git branch -r --merged main | grep -v main
```

**Delete a remote branch:**
```bash
git push origin --delete branch-name
```

**Delete a local branch:**
```bash
git branch -d branch-name  # Safe delete (only if merged)
git branch -D branch-name  # Force delete (use with caution)
```

### Stale Branch Policy

**Definition of stale:** No commits for >30 days

**Monthly review:**
```bash
# List branches by last commit date
git for-each-ref --sort=-committerdate refs/remotes/origin/ \
  --format='%(committerdate:short) %(refname:short) %(subject)'

# Review and delete stale branches
```

## 📊 Branch Monitoring

### Check Branch Status

**List all branches:**
```bash
git branch -a
```

**Check if branch is merged:**
```bash
git branch -r --merged main | grep your-branch-name
```

**Count commits ahead/behind:**
```bash
git rev-list --count origin/main..origin/your-branch
```

**See branch graph:**
```bash
git log --all --graph --oneline --decorate
```

### Health Check Commands

Add these to your regular workflow:

```bash
# Weekly: Check for stale branches
git for-each-ref --sort=-committerdate refs/remotes/origin/ \
  --format='%(committerdate:short) %(refname:short)' | head -20

# Before starting work: Ensure main is current
git checkout main && git pull origin main

# After finishing work: Check what's not merged
git branch -r --no-merged main
```

## 🚀 Best Practices

### DO ✅

1. **Branch from main** - Always create new branches from up-to-date main
2. **Push regularly** - Push commits frequently to backup work
3. **Keep branches focused** - One feature/fix per branch
4. **Update from main often** - Rebase or merge main regularly to avoid conflicts
5. **Delete after merge** - Clean up merged branches immediately
6. **Use descriptive names** - Make branch purpose clear from name
7. **Write good commit messages** - Help future you understand changes

### DON'T ❌

1. **Don't force push to main** - Ever. Seriously.
2. **Don't work directly on main** - Always use a branch
3. **Don't keep stale branches** - Delete merged branches promptly
4. **Don't use generic names** - Avoid names like "test", "temp", "fixes"
5. **Don't accumulate branches** - Keep repository clean
6. **Don't commit secrets** - Use .env files and .gitignore
7. **Don't commit to someone else's branch** - Unless collaborating

## 🔧 Troubleshooting

### "Branch already exists"
```bash
# Delete old branch first
git branch -D branch-name
git push origin --delete branch-name

# Create new one
git checkout -b branch-name main
```

### "Merge conflicts"
```bash
# Update from main
git fetch origin main
git rebase origin/main

# Resolve conflicts in files
# Then:
git add .
git rebase --continue
```

### "Can't delete branch - not merged"
```bash
# Check if really merged
git branch -r --merged main | grep your-branch

# If truly not merged and you want to delete anyway:
git push origin --delete branch-name --force

# If merged but git doesn't recognize:
git branch -D branch-name  # Local
git push origin --delete branch-name  # Remote
```

### "Permission denied (403)"
For Claude Code branches, you can only modify branches with your current session ID. This is a security feature. Delete other branches manually via GitHub web interface.
