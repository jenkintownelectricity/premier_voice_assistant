💡 How Your Stack Works

You don't host LiveKit yourself. LiveKit Cloud is a managed WebRTC service that handles all the real-time audio/video infrastructure. Your Railway Worker runs the AI agent code that connects to LiveKit Cloud. When a user joins a room, LiveKit notifies your worker, which then processes audio using Deepgram (STT), your LLM chain (Fast Brain/Groq/Claude), and Cartesia (TTS).

🔊 User Hears
1 User clicks "Start Call"
Frontend creates room via POST /livekit/rooms and connects to LiveKit Cloud via WebRTC
2 LiveKit notifies Worker
Railway worker receives job notification and joins the room automatically
3 Voice Activity Detection
Silero VAD detects when user starts/stops speaking
4 Speech → Text
Deepgram transcribes audio to text in real-time (~30ms)
5 LLM Processing
Fast Brain (or Groq/Claude) generates response (~50-100ms)
6 Text → Speech
Cartesia synthesizes audio and streams back to user (~30ms)
