💡 How Your Stack Works

You don't host LiveKit yourself. LiveKit Cloud is a managed WebRTC service that handles all the real-time audio/video infrastructure. Your Railway Worker runs the AI agent code that connects to LiveKit Cloud. When a user joins a room, LiveKit notifies your worker, which then processes audio using Deepgram (STT), your LLM chain (Fast Brain/Groq/Claude), and Cartesia (TTS).

2. WebRTC Connection

3. Agent Joins Room

Railway worker receives notification, joins room

4. Speech → Text

Deepgram Nova-2 transcribes (~30ms)

5. LLM Response

Fast Brain → Groq → Claude (~50-100ms)

6. Text → Speech

Cartesia Sonic streams audio (~30ms)

🔧 Common Issues & Fixes

❌ PEP 668 externally-managed-environment error
▼
Error: error: externally-managed-environment × This environment is externally managed Cause: Nix-managed Python blocks direct pip installs.

Fix: Update nixpacks.toml to use a virtual environment: [phases.install] cmds = [ "python -m venv /app/venv", "/app/venv/bin/pip install --upgrade pip", "/app/venv/bin/pip install -r requirements.txt", "/app/venv/bin/pip install 'livekit-agents[deepgram,cartesia,openai,silero]==1.1.7'" ] [start] cmd = "/app/venv/bin/python backend/livekit_worker.py start"
❌ cannot import name 'room_io' from 'livekit.agents'
▼
Error: Failed to import LiveKit SDK: cannot import name 'room_io' from 'livekit.agents' Cause: Old version of livekit-agents installed (pre-v1.0).

Fix: Force install v1.1.7 with all extras: pip install 'livekit-agents[deepgram,cartesia,openai,silero]==1.1.7'
⚠️ Agent not responding / Silent calls
▼
Possible causes:
  • LLM fallback chain failed (all 3 LLMs unavailable)
  • Missing GROQ_API_KEY or ANTHROPIC_API_KEY
  • Fast Brain URL not responding
Check: # Check Railway worker logs for: "No LLM configured or available" "LLM fallback chain exhausted" # Verify env vars are set: echo $GROQ_API_KEY echo $ANTHROPIC_API_KEY
⚠️ git push rejected (non-fast-forward)
▼
Error: ! [rejected] main -> main (non-fast-forward) Cause: Your local branch is behind remote.

Fix: git pull origin main git merge origin/claude/agents-dashboard-qSsVa git push origin main Or force push (if you're the only contributor): git push origin main --force
ℹ️ High latency (>500ms voice-to-voice)
▼
Optimize:
  • Use Groq (faster than Claude) as primary LLM
  • Enable prompt caching with Claude
  • Check Modal cold starts - increase keep-warm
  • Use streaming for all services
