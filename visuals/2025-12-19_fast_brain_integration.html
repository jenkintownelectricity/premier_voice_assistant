Fast Brain Integration

HIVE215 + Fast Brain LPU - Dual-System AI for Sub-Second Voice

Dual-System Architecture

Fast Brain uses Kahneman's "Thinking, Fast and Slow" architecture - two systems that automatically route queries based on complexity.

System 1 - Fast Brain

~80ms
Groq LPU + Llama 3.3 70B
90% of queries - Simple questions, greetings, FAQs

System 2 - Deep Brain

~2000ms
Claude 3.5 Sonnet
10% of queries - Complex analysis, calculations

The Filler Phrase Innovation

When System 2 is needed, Fast Brain returns a filler phrase first. This filler plays via TTS (~2-3s) while Claude processes, creating natural conversation flow.

User: "Analyze my 850 kWh bill"
→
Fast Brain routes to System 2
→
"Let me pull up that info..."
→
Full response plays after filler
Result: Perceived latency is sub-second even when using Claude for complex queries.

API Endpoints

POST /v1/chat/hybrid
Recommended for voice. Auto-routes between System 1 and System 2. Returns filler phrase when System 2 is used.
POST /v1/chat/voice
Returns text + voice hints for TTS systems like Parler that can interpret voice descriptions.
GET /v1/greeting/{skill_id}
Get skill-specific greeting text and voice description.
GET /v1/fillers
Get all filler phrase categories (analysis, calculation, research, default).
GET /v1/skills
List all available skill adapters.
POST /v1/skills
Create a custom skill with system prompt and knowledge base.
GET /health
Health check with architecture info (System 1/2 models, latency, skills list).

Integration Code

Using the Hybrid Endpoint

# Recommended pattern for voice agents
response = await brain.hybrid_chat(
    messages=[{"role": "user", "content": user_input}],
    skill="electrician",
    user_context={"business_name": "Jenkintown Electricity"}
)

# If filler was returned (System 2 was used), play it first
if response.filler:
    await tts.speak(response.filler)  # Plays while Claude thinks

# Then play the main response
await tts.speak(response.content)

Response Format

{
  "content": "Based on your 850 kWh usage at $0.12/kWh, your bill should be...",
  "filler": "Let me pull up your information and analyze that...",  // null if System 1
  "system_used": "deep",  // "fast" or "deep"
  "fast_latency_ms": 85,
  "deep_latency_ms": 1950,
  "skill_used": "electrician"
}

Environment Variables

Variable Required Description
FAST_BRAIN_URL Yes Fast Brain Modal endpoint (e.g., https://username--fast-brain-lpu-fastapi-app.modal.run)
DEFAULT_SKILL No Default skill adapter (default: receptionist)
GROQ_API_KEY Fallback Groq API key for fallback if Fast Brain unavailable
Fallback Chain: Fast Brain → Groq → Anthropic. If Fast Brain is down, the worker automatically falls back to Groq direct.
