Common Dependency Errors & Fixes

cannot import name 'room_io' from 'livekit.agents'

The room_io module was restructured in newer versions.

Fix: Update livekit-agents to 1.3.8
pip install 'livekit-agents[deepgram,cartesia,openai,silero]==1.3.8'

ResolutionImpossible: opentelemetry-api conflict

livekit-agents 1.3.8 requires opentelemetry-api>=1.39.0, but you pinned 1.21.0

Fix: Update requirements.txt
Change opentelemetry-api==1.21.0 to opentelemetry-api>=1.39.0

pip backtracking (19+ minutes)

Unpinned livekit plugin versions cause pip to try many combinations.

Fix: Pin ALL livekit plugins to the same version
All plugins must be ==1.3.8 (matching livekit-agents)

uvicorn: command not found

Nixpacks venv not activated, or using nixpacks instead of Dockerfile.

Fix: Use Dockerfile deployment (Option B), not nixpacks
Delete nixpacks.toml, ensure Dockerfile exists

AttributeError: type object 'LLM' has no attribute 'with_groq'

Old Groq API syntax in livekit-plugins-openai.

Fix: Use OpenAI-compatible API:
llm = openai.LLM(api_key=GROQ_KEY, base_url="https://api.groq.com/openai/v1")

Version History & Lessons Learned

Current Version - December 18, 2025
Works with room_io API, requires opentelemetry>=1.39.0. All plugins must match this version.
1.1.7
BROKEN - December 17, 2025
Missing room_io module. Import error: "cannot import name 'room_io'". Worked with old VoicePipelineAgent API but that's deprecated.

OpenTelemetry CONFLICT
Pinning opentelemetry to 1.21.0 breaks livekit-agents 1.3.8. ResolutionImpossible error during pip install.
