Docker Build Optimization

Why your builds went from minutes to seconds

Build Time: Before vs After

Before (No Layer Caching)

~5-10 min

Every build reinstalls all dependencies

After (Layer Caching)

~20 sec

Only changed layers rebuild

The Secret: Layer Ordering

💡

Docker caches layers top-to-bottom

If a layer changes, all layers BELOW it must rebuild. So put slow, stable things (like pip install) at the TOP, and fast, changing things (like your code) at the BOTTOM.

# Dockerfile - Optimized Layer Order

FROM python:3.11-slim           # Layer 1: Base (cached)

WORKDIR /app                     # Layer 2: Setup (cached)

RUN apt-get update && ...        # Layer 3: System deps (cached)

COPY requirements.txt .          # Layer 4: JUST requirements (cached if unchanged)
RUN pip install -r requirements.txt  # Layer 5: Slow pip install (CACHED!)

RUN python -c "..."              # Layer 6: Download models (cached)

COPY . .                         # Layer 7: Your code (rebuilds on every push)

CMD ["bash", "scripts/start.sh"] # Layer 8: Start command

The "0ms" in Railway logs

When you see "cached" next to a step, Docker is reusing the layer from a previous build. It doesn't re-run the command at all - just uses the cached result. That's why pip install shows "0ms" even though it normally takes 3+ minutes.

What Triggers a Full Rebuild?

Change Type Layers Rebuilt Time
Python code change Only COPY . . ~20 sec
requirements.txt change pip install + below ~5 min
Dockerfile change Everything below change ~5-10 min

Optimizations Applied

1

Copy requirements.txt separately

Before: COPY . . then pip install (always rebuilds)

After: COPY requirements.txt . then pip install (cached if unchanged)

2

Use python:3.11-slim

Smaller base image = faster download + less to cache

3

Direct model download

Instead of running the full worker script (which has complex imports), use a minimal Python command to download the turn detector model

4

Clean apt cache

rm -rf /var/lib/apt/lists/* removes package lists after install, reducing layer size

Your Optimized Dockerfile

FROM python:3.11-slim

WORKDIR /app

# System dependencies (cached unless Dockerfile changes)
RUN apt-get update && apt-get install -y --no-install-recommends \
    ffmpeg \
    libopus0 \
    && rm -rf /var/lib/apt/lists/*

# Python dependencies (cached unless requirements.txt changes)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
RUN pip install --no-cache-dir 'livekit-agents[deepgram,cartesia,openai,silero]==1.3.8'

# Download turn detector model (cached)
RUN python -c "from livekit.plugins.turn_detector import EOUModel; EOUModel().initialize()" || echo "skipped"

# Application code (rebuilds on every push - BUT THAT'S FAST!)
COPY . .

CMD ["bash", "scripts/start.sh"]
